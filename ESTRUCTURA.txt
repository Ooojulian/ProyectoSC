📅 PARA EL MARTES (4 COMPUTADORES)
═══════════════════════════════════════════════════════════════════════════════

PASO 1 - EN CADA COMPUTADOR (configurar red):
  $ sudo /home/julian/ProyectoSC/scripts/setup_adhoc.sh ProyectoSC 6

PASO 2 - EN CADA COMPUTADOR (cargar BATMAN):
  $ sudo modprobe batman-adv
  $ sudo ip link add name bat0 type batadv
  $ sudo ip link set wlan0 master bat0
  $ sudo ip addr add 192.168.200.10X/24 dev bat0  (X=1,2,3,4)

PASO 3 - EN CADA COMPUTADOR (ejecutar sensor):
  $ NODE_NAME="Node-X" python3 sensor_node.py &
  $ NODE_NAME="Node-X" python3 web_dashboard/app.py &

PASO 4 - ACCEDER A DASHBOARDS:
  Node 1: http://192.168.200.101:5000
  Node 2: http://192.168.200.102:5000
  Node 3: http://192.168.200.103:5000
  Node 4: http://192.168.200.104:5000

📊 MÉTRICAS QUE CALCULA
═══════════════════════════════════════════════════════════════════════════════

τ (Latencia)           = Tiempo desde creación hasta almacenamiento (ms)
ρ (Utilización)        = λ/μ (factor de ocupación del sistema)
W(q)                   = Tiempo promedio en cola (fórmula M/M/1)
L(q)                   = Paquetes promedio en cola
PLR (Packet Loss Rate) = Paquetes perdidos / total enviados
Throughput             = Bytes procesados por segundo

✨ CONCEPTOS DEMOSTRADOS
═══════════════════════════════════════════════════════════════════════════════

1. EMERGENCIA
   → Comportamiento global emerge de interacciones locales
   → Sin coordinador central, el sistema se auto-sincroniza

2. AUTO-ORGANIZACIÓN
   → BATMAN recalcula rutas automáticamente
   → Topología se adapta a cambios de ambiente

3. ADAPTACIÓN
   → Se recupera de nodos caídos
   → Redirige tráfico automáticamente

4. NO LINEALIDAD
   → Performance no escala linealmente con nodos
   → Latencia aumenta exponencial con carga

5. RESILIENCIA
   → Mientras haya 1 nodo activo, el sistema funciona
   → Tolerancia a pérdida de hasta N-1 nodos
